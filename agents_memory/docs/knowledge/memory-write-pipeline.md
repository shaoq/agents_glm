## 0. 核心认知

记忆写入管线的本质，是把「**给人看的对话**」转成「**机器可高效召回的个性化记忆**」。三个关键判断：

1. **不是所有交互都值得记忆**：闲聊 / 寒暄 / 工具执行细节不应写入，只记可复用事实。
2. **写入质量决定召回上限**：写入的噪音 / 重复 / 矛盾，会直接污染后续召回（garbage in, garbage out）。
3. **写入是主动决策，不是被动存档**：每条候选记忆要经过「抽取 → 查重 → 冲突判断 → 存储」四步决策链。

---

## 1. 全景：写入管线的 4 个环节

```
交互消息 → ①事实抽取(Fact Extraction) → ②查重(Context Lookup)
        → ③冲突判断(ADD/UPDATE/NOOP) → ④存储(Memory Store)
```

一条主线：从原始对话到持久化记忆的完整流水线。下文逐环节展开。
