# Agent Memory 知识点：记忆维护管线（Memory Maintenance Pipeline）

> **文档定位**：本文聚焦 Agent Memory 的**维护管线**——记忆的遗忘与反思，让记忆库随时间保持精炼、高价值。
>
> **为什么需要维护**：
> - 遗忘：记忆无限增长 → 噪音淹没信号 → 召回质量下降
> - 反思：原始记忆碎片化 → 缺乏高阶洞察 → Agent 无法从经验中学习
>
> **深入进度**：⏳ 大纲，随讨论逐环节深入
>
> **更新日期**：2026-07-24

## 0. 核心认知

记忆维护管线的本质，是让记忆库「**像人脑一样遗忘无关的、强化重要的**」。三个关键判断：

1. **遗忘不是删除，是降权**：不重要的记忆不物理删除，而是降低召回权重（access_count 衰减 / importance 衰减）。
2. **反思是记忆的升华**：从多条碎片记忆综合出高阶洞察（「用户正在从 RAG 转向 Memory 开发」），反思记忆 importance 更高。
3. **维护是离线/异步的**：不阻塞实时交互（定时 / 累积触发，类似 agents_rag 的离线索引）。

---

## 1. 全景：维护管线的 2 个子管线

```
─── 遗忘子管线（Forgetting）───
[定时触发] → 扫描记忆库 → 评估每条记忆 → 降权/淘汰/过期清理

─── 反思子管线（Reflection）───
[累积 importance 超阈值触发] → 检索近期记忆 → LLM 综合高阶洞察 → 写入高 importance 记忆
```

## 2. 遗忘策略（Forgetting Policies）⭐

> ⏳ 待深入

- **Importance 衰减**：随时间推移降低 importance 分数（`importance *= decay_factor`）
- **LRU 淘汰**：最久未访问的记忆降权 / 移除
- **Session 级过期**：session 结束后自动过期临时记忆
- **显式删除**：用户/Agent 主动 delete（隐私 / 纠错）
- **摘要替代**：多条细节 → 摘要为一条高阶记忆（压缩冗余）
- 策略组合（分层：session 级 TTL + 长期 importance 衰减 + 显式 delete）

## 3. 反思（Reflection）⭐⭐

> ⏳ 待深入

- **触发**：近期记忆 importance 累积超阈值（Stanford Generative Agents 模式）
- **提问**：LLM 针对近期记忆生成反思问题（「用户最近在关注什么？」）
- **检索**：用反思问题检索相关历史记忆
- **综合**：LLM 将碎片记忆综合为高阶洞察
- **写入**：反思结果作为新记忆写入（importance 更高）
- 程序记忆更新（LangGraph 模式：反思优化 prompt → 存回）
- 定时 vs 累积触发（权衡）
